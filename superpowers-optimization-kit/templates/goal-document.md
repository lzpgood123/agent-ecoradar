## 阶段流程

每个阶段结束必须通过 `gatekeeper` skill 派 `delegate_task` 独立子 Agent 校验；执行 Agent **不得自检放行**。FAIL 则调用 `auto-recovery`。

### 阶段 1：Intake / 决策编译

1. 加载 `completion-contract` skill，初始化/确认完成契约活文档
2. 将 PRD 目标全部写入契约；为每项目标至少挂 1 条验收场景
3. 填充决策矩阵；处理 open 矛盾（必要时 `decision-arbiter`）
4. 冻结验收场景（`冻结=是`）
5. 门禁：`gatekeeper` → 阶段 Intake → 报告写入 `docs/superpowers/gates/`

### 阶段 2：Spec + Plan

1. 产出/更新 spec 与可执行 plan（路径写入契约证据）
2. 核对与「不可改边界」无冲突
3. 门禁：`gatekeeper` → Spec+Plan

### 阶段 3：实现

1. 按 plan 逐步实现；真实跑测试并保留输出
2. 不扩大范围到非目标
3. 门禁：`gatekeeper` → 实现

### 阶段 4：验收证据

1. 按契约「验收场景」逐条实测，写入证据路径
2. 证据类型符合任务（后端：curl/测试日志；前端：截图/E2E；运维：命令输出）
3. 门禁：`gatekeeper` → 验收证据

### 阶段 5：Wiki / 合规

1. 按变更范围更新应改的 wiki（L4B/L5 等）
2. 加载 `wiki-checkpoint`，合规通过后把报告路径写入契约
3. 门禁：`gatekeeper` → Wiki/合规

### 阶段 6：完成门

1. 加载 `completion-contract`，执行硬门槛四项检查
2. 仅当 4/4 全绿时将「可否宣布完成」改为「是」
3. 门禁：`gatekeeper` → 完成门
4. 向用户/goal judge 显式声明完成（供 `/goal` 判 done）

## 失败恢复策略

1. 门禁 FAIL / 测试失败 / 实现失败 → 加载 `auto-recovery`
2. 读取完成契约「已试方法」；新策略摘要不得与已试相同
3. PRD 矛盾/缺口 → `decision-arbiter`（写入 P1）；环境/权限缺失 → 写失败报告并停止扩 scope
4. 策略穷尽 → 按 `templates/failure-report.md` 写到 `docs/superpowers/failures/`，通知用户，不阻塞其他任务
5. 禁止原地重复同一失败方法；禁止跳过门禁
