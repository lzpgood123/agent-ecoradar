# Meta 验收剧本 — 连续 3 次端到端全绿

> 对应 plan 任务 13 与设计决策 D18–D19。  
> **Meta 系统做成标准：** 同一目标项目上，3 份不同 PRD，覆盖前端 + 后端，每次完成契约硬门槛全绿，运行期无人干预（仅允许 pre-grill 人机循环）。

## 前提

1. 目标项目已安装本优化包（`superpowers-optimizer` 或按 `GUIDE.md` 安装）
2. `docs/superpowers/{goals,contracts,gates,failures}/` 已创建
3. Hermes 支持 `/goal`；建议 `goals.max_turns` ≥ 100
4. 用户仅在 pre-grill 阶段参与（可行性 / 准备清单 / 确认 Goal）

## 运行规则

| 规则 | 要求 |
|------|------|
| 项目 | 3 次必须在**同一目标项目** |
| PRD | 3 份**各不相同** |
| 类型覆盖 | 至少 1 次前端 + 1 次后端（第 3 次可为前端/后端/全栈） |
| 完成 | 每次完成契约 4 项硬门槛全绿 |
| 干预 | 运行期禁止人工改代码代跑；pre-grill 可循环改 PRD |
| 门禁 | 必须 `delegate_task` 独立校验，禁止执行 Agent 自放行 |
| PRD 主权 | 不可改 PRD 目标；矛盾走 `decision-arbiter` |

## 建议的 3 次 PRD 主题（可替换）

### Run A — 后端

- 示例：健康检查 / 只读 API / 日志字段
- 验收证据：测试 + `curl`/API 实测

### Run B — 前端

- 示例：列表筛选 / 空状态文案 / 可点击交互
- 验收证据：可点可验截图或交互日志（拒绝摆设 UI）

### Run C — 前端或后端（与 A/B 不同）

- 示例：小范围全栈联调或另一端独立功能
- 验收证据：按任务类型动态

## 失败处理

- 任一次未全绿：记录 failures/ 报告，修复优化包或项目环境后**从失败的那次重跑**（连续计数重置或按团队约定从断点重计；推荐：失败后重置连续计数）。
- 仅 pre-grill 改 PRD 不算运行期干预。
