# L5-接口契约 — 数据怎么流

> 前后端协作的"合同"。前后端 Agent 写代码时以此为准，不一致就是 bug。

## 前后端通信方式

纯静态站点模式。前端不调用任何后端 API，所有数据预先构建为 JSON 文件放在 site/data/ 目录下。

前端通过 `fetch('data/<filename>.json')` 加载。

## 数据文件清单

| 文件 | 内容 | 前端使用 |
|------|------|---------|
| data/projects.json | 全量 618 条项目 | 主数据源 |
| data/curated-projects.json | 60 条推荐 | curatedOnly 筛选 |
| data/rejected-projects.json | 25 条拒绝 | 仅在 index 展示 |
| data/tools.json | 11 个工具定义 | 工具筛选下拉框 |
| data/metrics.json | 统计摘要 | 页头度量卡片 |
| data/concepts.json | 10 个概念定义 | 当前前端未使用 |
| data/i18n.json | 分类翻译 | 当前前端未使用 |

## 数据生成源

所有 site/data/*.json 由 `scripts/build_site.py` 从以下源生成：
1. data/projects.yaml → projects.json + metrics JSON
2. data/curated-projects.yaml → curated-projects.json
3. data/rejected-projects.yaml → rejected-projects.json
4. data/seed-tools.yaml → tools.json
5. data/concepts.yaml → concepts.json

## Curated 项目额外字段

相比 Project 对象增加：
```
"recommendation_level": "try-now|watch|reference",
"curation_note": "Auto-selected by scoring rules: source=github, score=18, categories=skills-prompts, mcp-acp-a2a."
```

## Rejected 项目额外字段

相比 Project 对象增加：
```
"rejection_reason": "Auto-rejected by scoring/source-quality rules: low confidence..."
```

## 数据流对照

| 前端操作 | 数据来源 | 渲染内容 |
|---------|---------|---------|
| 页面加载 → 统计卡片 | metrics.json | 总记录、推荐、拒绝数等 |
| 页面加载 → 官方工具卡片 | projects.json (ranking_scope=official) | 10 个官方工具卡片 |
| 搜索/筛选/排序 | projects.json（按条件过滤） | 项目列表表格 |
| 只看推荐 | projects.json（id 交集） | 仅 curated 项目 |
| 双语切换 | 项目本身 i18n 字段 | 字段名和摘要中英文 |
| 点击详情 | 当前选中项目 | 评分明细 + notes |
| 点击报告链接 | site/reports/*.md | 浏览器直接打开 |
