# L5-接口契约 — 数据怎么流

> 前后端协作的"合同"。管道脚本 ↔ 站点数据文件 以此为准，不一致就是 bug。

## 请求格式

管道采用"文件传递"模式，无 HTTP 请求。每个脚本读取 data/ 目录下的文件，处理后写入新文件。

**加载方式：** `load_jsonish(rel)` — 从项目根目录加载 JSON/YAML 文件
**保存方式：** `save_jsonish(rel, data)` — 保存 JSON 到文件，自动创建目录

## 响应格式

管道以文件系统的状态变化作为响应。关键输出文件：

- `data/projects.yaml` — 所有归一化记录
- `data/curated-projects.yaml` — 推荐集
- `data/rejected-projects.yaml` — 低质/噪声集
- `data/scores.yaml` — 评分数据
- `site/data/projects.json` — 站点用数据
- `site/data/curated-projects.json` — 站点用推荐数据

## 端点清单

### 站点 JSON API（公开）

| 端点 | 路径 | 数据 | 更新频率 |
|------|------|------|---------|
| 项目数据 | /data/projects.json | 所有归一化记录 | 每日更新 |
| 推荐数据 | /data/curated-projects.json | 60 条 curated 记录 | 每日更新 |
| 拒绝数据 | /data/rejected-projects.json | 25 条 rejected 记录 | 每日更新 |
| 工具数据 | /data/tools.json | 10 个目标工具定义 | 极少变更 |
| 概念数据 | /data/concepts.json | 分类体系 | 极少变更 |
| 指标数据 | /data/metrics.json | 统计数据 | 每日更新 |
| 多语言数据 | /data/i18n.json | 中英双语文本 | 每日更新 |

## 数据流对照

| 前端操作 | 对应数据 | 来源 | 展示方式 |
|---------|---------|------|---------|
| 页面加载 | projects.json + curated-projects.json + tools.json | site/data/ | 渲染主表格 |
| 分类筛选 | projects[i].category | 每条记录 | 下拉过滤 |
| 工具筛选 | projects[i].target_tools | 每条记录 | 下拉过滤 |
| 详情查看 | i18n.zh/en → name + summary | 每条记录 | 表格行数据 |
| 双语切换 | i18n.zh 或 i18n.en | 每条记录 | 重新渲染 |
| 排序 | total_score / name / last_seen | 每条记录 | 表格重排 |
