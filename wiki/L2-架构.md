# L2-架构 — 系统怎么搭的

> 读完知道"系统怎么搭的"。技术选型、模块划分、数据流的全景图。

## 系统架构

```
采集层（Collectors）
  ├─ collect_github.py  → data/raw/github/<date>/
  ├─ collect_exa.py     → data/raw/exa/<date>/
  └─ collect_web.py     → data/raw/web/<date>/（降级兜底）
          │
          ▼
归一化层（Normalize & Enrich）
  ├─ normalize.py       → data/projects.yaml（统一字段、去重、分类）
  ├─ enrich_i18n.py     → 填充 i18n.zh/en 结构
  └─ validate_data.py   → 数据完整性验证
          │
          ▼
评分与定稿层（Score & Finalize）
  ├─ score.py           → data/scores.yaml（6 维评分 + 权重调整）
  └─ finalize_data.py   → data/curated-projects.yaml + rejected-projects.yaml
          │
          ▼
报告与站点层（Reports & Site）
  ├─ generate_reports.py  → docs/reports/*.md（10+ 种报告）
  ├─ build_site.py        → site/data/*.json + site/index.html
  ├─ quality_gate.py      → PASS/FAIL 最终验证
  └─ deploy_site.py       → → /var/www/coding.lzpgood.online/（Nginx webroot）
          │
          ▼
版本控制层（GitHub）
  ├─ bump_version.py    → VERSION + CHANGELOG.md
  ├─ release_notes.py   → GitHub Release
  └─ GitHub Actions     → Update Data / Publish Site / Create Release
```

## 技术选型

| 技术 | 选型理由 |
|------|---------|
| Python 3.12+ | 脚本化数据管道，标准库丰富，适合采集/处理/构建任务 |
| GitHub CLI (gh) | GitHub 搜索和仓库数据采集，无需额外 API key |
| Exa (via mcporter MCP) | 语义搜索引擎，比传统搜索更适合发现 AI 生态资料 |
| Bing RSS (fallback) | Exa 不可用时的降级兜底，明确标注 fallback-not-exa |
| YAML (data 文件) | 人类可读、Git 友好的数据格式 |
| JSON (site 数据) | 机器可读、浏览器原生支持 |
| Nginx | 高性能静态文件服务器，支持安全头配置 |
| Let's Encrypt | 免费自动 TLS 证书 |
| GitHub Actions | 仓库侧自动化，workflow_dispatch + cron |
| Hermes Agent | 生产服务器 cron 调度和部署 |
| mcporter | Exa MCP 协议接入层 |
| superpowers-zh 框架 | 23 个 skills 的中文增强框架，wiki 优先文档体系 |

## 数据流全景

1. **采集：** 每天 03:00 Hermes cron 触发 update_tracker.py → collect_github.py（gh search）+ collect_exa.py（mcporter Exa）→ 原始数据写入 data/raw/{github,exa}/<date>/
2. **归一化：** normalize.py 读取原始 JSON → 统一为标准记录（30+ 字段）→ 自动分类（14 类）、关联目标工具、去重 → data/projects.yaml
3. **丰富：** enrich_i18n.py 补齐 zh/en 展示结构 → validate_data.py 校验必填字段
4. **评分：** score.py 读取 config/scoring.yaml → 计算 6 维评分 + 来源/类别权重 + 惩罚 → data/scores.yaml
5. **定稿：** finalize_data.py 按排名阈值 → curated（60 条 top）/ rejected（25 条低质）/ official-seed（10 条）
6. **报告：** generate_reports.py 生成 10+ 份 Markdown 报告 → docs/reports/
7. **构建：** build_site.py 从 data/ 生成 site/data/*.json + site/reports/ → site/
8. **验证：** quality_gate.py 执行 15+ 项检查 → PASS/FAIL
9. **部署（可选）：** deploy_site.py → 生产 Nginx webroot
10. **推送：** git commit + push → GitHub Actions（Publish Site + Create Release）

## 部署架构

- **生产服务器：** 腾讯云上海服务器（Hermes Agent 环境）
- **Web 服务：** Nginx + Let's Encrypt TLS
- **部署路径：** /var/www/coding.lzpgood.online/
- **启动命令：** `python3 scripts/update_tracker.py --deploy --github-limit 30 --exa-limit 3`
- **自动化调度：** Hermes cron（每日 03:00）+ GitHub Actions（每天 UTC 02:00）
- **环境要求：** Python 3.12+、gh CLI（已认证）、mcporter（Exa MCP 接入）

## 关键边界

- **服务器侧（Hermes cron）：** 生产运行 + 部署，可访问真实 GitHub API 和 Exa
- **GitHub Actions 侧：** 仓库侧验证，无 mcporter（Exa 采集会失败，但非阻塞），无部署能力
- **正式站点（coding.lzpgood.online）：** 公开展示面，由 deploy_site.py 部署
- **GitHub Pages（giodio.me/search-in-coding/）：** 预览版，由 Publish Site workflow 部署
- **GitHub 总仓库：** 完整数据与历史，含 raw/ 和内部文档
