# L6-经验录 — 踩过什么坑

> 所有 Agent 的"避坑指南"。记录设计决策、已知问题、调试技巧、教训。

## 关键设计决策

1. **GitHub 是总仓库，正式站点是展示面**
   - 所有 raw 数据、归一化数据、报告、站点数据、版本记录都必须回写 GitHub
   - 正式站点只展示当前构建结果
   - 服务器 Hermes cron 负责生产运行，GitHub Actions 负责仓库侧验证

2. **全自动评分优先，取消人工审核**
   - curated 不再代表人工审核，而是自动推荐集
   - 自动进入优先级：GitHub verified > Exa verified > 高分 MCP/skills/rules > 各工具覆盖

3. **日期版本号迭代**
   - 版本号格式 YYYY.MM.DD
   - 重要更新自动 bump 和发布

4. **6 维评分体系**
   - ecosystem_value、activity、adoption、practicality、novelty、confidence
   - 来源权重：github=2、exa=1、official-seed=0、fallback-web=-2
   - 分类权重：mcp-acp-a2a/skills-prompts/rules-instructions/context-engineering=2

5. **多源独立失败策略**
   - 采集器可独立失败（GitHub 失败不影响 Exa，反之亦然）
   - 验证和构建质量门禁必须通过
   - 部署默认关闭（--deploy opt-in）

6. **来源透明原则**
   - Exa、GitHub、fallback-web 必须清楚标注
   - fallback-web 数据永不伪装成 Exa 结果，标记 fallback-not-exa

7. **可复用性设计**
   - 替换 seed-tools、queries、concepts、scoring 配置即可迁移到其他技术生态

## 已知坑

1. **Exa 依赖问题（2026-07-06）**
   - 现象：Exa 通过 mcporter 接入，如果 mcporter 未配置或 MCP server 不可用，Exa 采集会失败
   - 原因：mcporter 是独立进程，需要确认其运行状态
   - 解决方案：保留 fallback-web 降级通道，所有 fallback 记录标注 fallback-not-exa

2. **数据质量波动（2026-07-06）**
   - 现象：fallback-web 来源包含大量噪声（如宠物广告、不相关页面）
   - 原因：Bing RSS 搜索精度不足，无语义理解
   - 解决方案：自动 rejected 集过滤明显不相关记录，评分惩罚 fallback 来源

3. **分类过度集中（2026-07-06）**
   - 现象：skills-prompts 和 testing-review-ci 占比过高，部分工具分类分布不均
   - 原因：分类规则偏宽，关键词匹配精度不足
   - 解决方案：无短期方案，需优化 CATEGORY_RULES 关键词

4. **双语言不是真翻译（2026-07-06）**
   - 现象：i18n 字段是结构化 fallback，不是真正翻译
   - 原因：管道无 LLM 调用，所有处理为确定性规则
   - 解决方案：当前可接受，未来可引入 LLM 翻译

5. **GitHub Actions 中 Exa 不可用（2026-07-06）**
   - 现象：GitHub Actions 环境没有 mcporter，Exa 采集返回空
   - 原因：Actions runner 无 mcporter 配置
   - 解决方案：非阻塞设计，Exa 采集失败不影响其他步骤

6. **仓库膨胀风险（2026-07-06）**
   - 现象：raw 数据长期直接入 Git 导致仓库变大
   - 原因：每日采集保留原始 JSON
   - 解决方案：archive_raw.py 超期归档策略，但需定期执行

## 调试技巧

- **启动管道：** `python3 scripts/update_tracker.py --skip-collect`（跳过采集，只处理已有数据）
- **完整运行：** `python3 scripts/update_tracker.py --github-limit 30 --exa-limit 3 --deploy`
- **查看日志：** 管道所有步骤输出 JSON 结构化日志到 stdout
- **运行测试：** `python3 -m pytest tests/ -v`
- **质量门禁：** `python3 scripts/quality_gate.py`
- **数据验证：** `python3 scripts/validate_data.py`
- **站点预览：** site/index.html 在浏览器中直接打开即可查看

## 常见错误

| 错误信息 | 原因 | 修复方法 |
|---------|------|---------|
| quality_gate.py FAIL: curated < 50 | 数据量不足 | 调整评分阈值或等待新采集 |
| collect_exa.py: mcporter not found | mcporter 未安装 | npm install -g mcporter |
| gh: not authenticated | GitHub CLI 未登录 | gh auth login |
| deploy_site.py: SEARCH_IN_CODING_WEBROOT not set | 环境变量未配置 | 设置环境变量或使用 --dest |
| quality_gate.py: review_state 过期 | 数据中有旧状态值 | 运行 normalize_project_fields() |
