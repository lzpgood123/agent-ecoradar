# L1-全景 — 项目是什么

> 所有人第一站。读完知道"这个项目是什么，它做什么，核心流程怎么走"。

## 产品定位

Search in Coding 是一个面向终端型 / Agentic AI Coding 工具的持续自动更新生态追踪索引库。

它不是普通的 awesome list，而是一个每天自动发现、归一化、评分、分类 AI Coding Agent 生态资源（工具本体、插件、MCP/ACP/A2A、skills、rules、prompts、上下文工程、PR review、CI 自动化、教程、案例和评测）的索引库。通过 GitHub 总仓库（完整数据与历史）和正式站点（公开展示面）双渠道交付，完全自动化运营，无需人工逐条审核。

**竞品对比：** 与普通 awesome list（手动维护、更新慢、无评分体系）和纯搜索引擎（无结构化、无分类、无推荐）不同，Search in Coding 实现了全自动采集 → 归一化 → 评分 → 推荐 → 发布的一体化管道。

## 核心流程

1. 用户访问 https://coding.lzpgood.online/ 查看当前 AI Coding 工具生态状态
2. 从 Claude Code、Codex CLI、Cursor、Hermes Agent 等 10 个目标工具入口进入，查看每个工具的生态扩展资源
3. 按 agent-harness、skills-prompts、mcp-acp-a2a、rules-instructions 等 14 个生态分类浏览
4. 查看自动评分的 curated 推荐项目榜（60 条），按 try-now / watch / reference / experimental 分级
5. 阅读自动生成的最终交付报告、工具生态对比、趋势与机会、来源质量审计等报告
6. 通过 JSON API（https://coding.lzpgood.online/data/）获取机器可读数据
7. 在生产服务器执行 `python3 scripts/update_tracker.py --github-limit 50 --exa-limit 5 --deploy` 手动触发更新

## 用户视角

1. **多源自动采集** — 从 GitHub（gh search/gh repo view）和 Exa（语义搜索，通过 mcporter MCP）自动采集 AI Coding 生态资源。支持 Bing RSS 降级兜底。
2. **数据归一化** — 将不同来源数据统一结构，自动分类到 14 个生态类别，关联到 10 个目标工具。
3. **6 维自动评分** — ecosystem_value、activity、adoption、practicality、novelty、confidence，加权计算总分，来源权重（github=2、exa=1、fallback=-2）。
4. **自动推荐** — 自动生成 curated（推荐集 60 条）和 rejected（低质 25 条），推荐等级 try-now/watch/reference。
5. **双语展示** — 每条记录持久化 i18n.zh/en 展示结构，站点支持中英切换。
6. **报告自动生成** — 10+ 种报告：最终交付、工具生态对比、趋势与机会、来源质量审计等。
7. **静态站点** — 从 data 自动构建静态站点，部署到生产域名（Nginx + TLS）。
8. **公开 JSON Data API** — 7 个端点机器可读数据。
9. **质量门禁** — quality_gate.py 自动验证记录数、字段完整性、i18n 完备性、review_state 合法性。
10. **版本管理** — 日期版本号（YYYY.MM.DD），自动 bump、CHANGELOG、GitHub Release。
