## 各功能详情

### F01 — 语义去重（TF-IDF baseline）

**优先级：** P1
**状态：** 规划中
**依赖：** 无

**概述：** 对 data/projects.yaml 中的记录进行基于文本相似度的去重检测。当前按 URL 去重，但同一项目可能有不同 URL 来源（GitHub + Exa 文章），需要语义去重。

**关键设计决策：** 
- 第一阶段使用 TF-IDF baseline（简单、无外部依赖）
- 永远不自动删除记录，只标记和降权
- 使用 new_records 与 existing_records 的对比检测

**实现要点：**
- 在 normalize.py 或 finalize_data.py 中添加去重逻辑
- 只对比 new_records 和 existing_records，避免 O(n^2)
- 标记重复记录（duplicate_of 字段或降分）

### F02 — 语义去重（embedding/FAISS）

**优先级：** P2
**状态：** 规划中
**依赖：** F01

**概述：** 将 TF-IDF baseline 升级为 sentence-transformers embedding + FAISS 向量索引。

**关键设计决策：** 
- 使用轻量模型（all-MiniLM-L6-v2）
- 自托管 Qdrant 作为可选增强

### F03 — 双语翻译增强

**优先级：** P1
**状态：** 规划中
**依赖：** 无

**概述：** 当前 i18n 字段是结构化 fallback（中英文放同一字段），需要升级为真正翻译。

**关键设计决策：**
- 导入阶段即持久化 i18n 字段到 data/projects.yaml
- build_site.py 只做防御性补齐，不再作为翻译来源
- 接入翻译 API（DeepL 或 Google Translate）
- 缓存翻译结果避免重复调用

### F04 — 评分规则细化

**优先级：** P1
**状态：** 规划中
**依赖：** 无

**概述：** 当前分类 agent-harness 和 testing-review-ci 偏高，需要调优。

**关键设计决策：**
- 调整 normalize.py 中的 CATEGORY_RULES 关键词
- 或增加排除规则避免过度匹配
- 调整 config/scoring.yaml 中的分类权重

### F08 — 测试覆盖增强

**优先级：** P1
**状态：** 规划中
**依赖：** 无

**概述：** 当前只有 2 个测试文件，覆盖不全。

**实现要点：**
- 为每个 scripts/*.py 添加参数化测试
- 测试正常路径 + 错误路径 + 边界条件
- 测试数据文件完整性
